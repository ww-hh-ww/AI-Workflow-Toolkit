# Governance Truth Review

Governance Truth asks whether AIWF makes the next Planner see reality. The tree
and records are not the mission; they are visibility. If they imply progress the
code and evidence do not support, they are dangerous.

## Tree Shape

- Does the Goal tree still describe mission capability decomposition?
- Are leaf Goals capabilities, not implementation details?
- Are there 1:1 Goal -> Plan mirrors?
- Is the tree shallow enough to navigate?
- Are orphan or overlapping Goals hiding unclear ownership?

## Alignment

- Do Goal, Plan, Task, and Milestone levels each do their own job?
- Are Plan dependencies real execution gates?
- Are Task dependencies capturing within-plan order?
- Does the Milestone represent a real capability increment?
- Does the visible structure hide deferred risks that Planner must disposition?

## State Drift

- Do ready, cancelled, closed, and open states match narrative docs?
- Do completed Tasks and Milestones still have evidence, testing, review, or
  architecture gaps?
- Would a fresh Planner choose the right next action from this tree and state?
